## Lecture notes - Stage 1 (computational thinking)

### What is coding?

- **Coding is** giving the computer **clear instructions it can follow, step by step**. It is not the same as only playing games, only drawing, or “fixing” the machine by turning it off and on.
- The computer does *exactly* what we tell it, in order.
- **Optional demo:** Write three steps to make a sandwich on the board. Then change the order — ask what goes wrong. That’s the same idea as code: **order matters.**

### Algorithms

- **An algorithm is** a **recipe** for solving a task: a **clear, ordered list of steps**.
- If you skip steps or jumble the order, you don’t get the result you wanted.
- **Link:** Same idea as **sequencing** below — we’ll use it again in Scratch and in Python later.

### Debugging

- **Debugging is** **finding and fixing** mistakes in your instructions or code.
- It’s not deleting your whole project, and it’s not changing brightness for fun — it’s **hunt the mistake, fix it, try again**.

### Sequencing

- **Sequencing means** the computer runs steps **in the order you wrote them**. Change the order → you often change the outcome.
- **Do together:** Three steps on the board (e.g. boil water → pour into cup → add tea). Scramble the order and ask: “Would you still get a safe cup of tea?”

### Patterns & repetition

- **Repetition (loops) is** doing the **same kind of action many times** without writing every line again.
- “Repeat the *same move* four times” is different from “four random different moves.”
- **Example:** Walking four sides of a square — same “move forward / turn” pattern, not four unrelated actions.

### Graph paper programming

- **Graph paper programming is** using **symbols or arrows** on a grid to stand for **movements or actions** — like a tiny program on paper.
- One wrong arrow and you land on the wrong square — that’s **sequencing** again, on a grid.

### Scratch — introduction

- **Scratch is** a **visual, block-based** language: you snap blocks together to build programs (stories, games, animations).
- It’s still **code** — the blocks are strict about what runs and in what order. The cat does what the script says.

## Lecture notes — Stage 2 (Scratch)

### Coordinates

- **Coordinates are** how we say **where** something is on the Scratch stage: **x** is **left–right**, **y** is **up–down**.
- **The middle of the stage is (0, 0)** — not necessarily the corner.
- **Do together:** Point to the centre. Move the sprite — **predict x and y** before you read the numbers.

### Conditionals

- **A conditional (`if … then … else`) is** “**if** this is true, do **this**; **otherwise** do **that**.” Two different paths from **one** test.
- We’re not running everything at once — we’re **choosing** a path based on a condition.

### Control blocks

- **Control blocks** are the ones that **control flow** — especially **repeat** and **forever** (loops).
- Which palette makes the computer **do something again and again**? That’s **Control**, not Sound or a single Motion block by itself.

### Events

- **Events** answer: “**When** does this script **start**?” — e.g. **when green flag clicked**, **when this sprite clicked**.
- This block is the **starter** — it’s not asking a question inside the game; it’s **firing** the script.

### Sensing

- **Sensing blocks** **ask questions about the project**: Is a key pressed? Is the sprite touching something? How far?
- **Events** start scripts. **Sensing** checks what’s true **while** the script runs. Put one **Event** hat and one **Sensing** block on the board and label them.

### Operators

- **Operators** do **maths and comparisons** in the moment: add, subtract, equals, greater than…
- Does this block **calculate** a value for a decision or a move? — that’s different from **storing** a value (**Variables**) or **repeating** work (**Control**).

### Variables

- **A variable is** a **name** for a value that can **change** while the program runs (e.g. score, lives).
- It’s not the project filename and it’s not a costume — it’s a **number or text we update**. Show the variable **on the stage** and change it in a loop.

### Custom blocks (My Blocks)

- **My Blocks** let you **make your own block** that runs a script you wrote — **reuse** the same behaviour in more than one place.
- Like packing several steps into one new word — later, text languages call that a **function**.

## Students who may need more focus

The app lists names when someone has **≥ 40% wrong** with **≥ 3** answers — open **Revision & report** for your session. While you support them, these three contrasts help:

- **Events** = what **starts** a script · **Sensing** = what we **check** while it runs.
- **Coordinates** — always **guess (x, y)** before moving the sprite.
- **Control** = repeat work · **Operators** = calculate a value (e.g. “repeat 10 times” vs “10 + 5”).
